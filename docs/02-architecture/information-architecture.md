# Information Architecture

## Purpose

Define how users navigate the platform before implementation begins.

---

# User Types

## Public Visitor

Can:

- View all public pages
- Explore projects
- Use AI features
- Download resume
- Contact me

Cannot:

- Access admin area

---

## Administrator

Can:

- Manage projects
- Manage resume
- Manage featured content
- Manage AI knowledge base (future)
- View analytics
- Configure platform settings

---

# Primary Navigation

- Home
- Projects
- Experience
- Resume
- About
- Contact

---

# AI Features

- AI Recruiter Assistant
- AI Resume Q&A
- AI Project Explainer
- AI System Design Walkthrough

---

# Admin

/ admin

Authentication required.

Admin pages are never publicly accessible.

---

# Future Modules

These are intentionally excluded from V1.

- Blog
- Analytics Dashboard
- AI Prompt Library
- AI Learning Timeline
- AI Playground
- Research Notes

---

# Success Criteria

A visitor should be able to reach any major feature within three clicks.

The navigation should remain stable even as new AI modules are added.
